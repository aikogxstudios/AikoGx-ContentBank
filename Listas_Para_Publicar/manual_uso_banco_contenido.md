# Manual sencillo - Como usar el banco de contenido

Este manual explica como usar este repositorio para sacar ideas de contenido sin liarse.

## 1. Para que sirve este repo

Este repo sirve para tener siempre ideas preparadas para:

- TikTok
- YouTube Shorts
- Discord
- X
- web/devlogs
- itch.io

La idea no es hacer videos perfectos. La idea es mantener el proyecto vivo y tener siempre algo que enseñar.

## 2. Flujo simple

1. Fak deja una nota del dia o una idea rapida.
2. Aiko convierte esa nota en ideas de contenido.
3. Las ideas se guardan en el banco.
4. El equipo mira la lista de esta semana.
5. Se elige una idea.
6. Se prepara el clip, captura, texto o voz.
7. Se publica.
8. Se marca como publicada o reciclable.

## 3. Que archivo mirar primero

Mirar primero:

`Listas_Para_Publicar/esta_semana.md`

Ese archivo dice que ideas son mejores para publicar ahora.

Despues mirar:

- `Listas_Para_Publicar/necesitan_clip.md` si hace falta grabar video.
- `Listas_Para_Publicar/necesitan_captura.md` si basta con una imagen.
- `Listas_Para_Publicar/reciclables.md` si no hay nada nuevo esa semana.
- `Banco_Ideas/ideas_tiktok_shorts.md` para buscar ideas de videos cortos.

## 4. Como elegir una idea

Elegir una idea que cumpla al menos una de estas cosas:

- se puede grabar rapido;
- enseña algo bonito;
- explica una parte del juego;
- hace una pregunta a la comunidad;
- muestra progreso real;
- es divertida o rara;
- sirve aunque no haya una gran novedad.

Si una idea necesita demasiado trabajo, dejarla para mas adelante.

## 5. Estados de las ideas

Cada idea puede tener un estado:

- Lista para preparar: ya se puede trabajar.
- Necesita clip: hay que grabar video.
- Necesita captura: hace falta una imagen.
- Necesita voz: hace falta narracion o audio.
- Necesita contexto: falta informacion para no publicar mal.
- Reciclable: se puede usar cuando no haya novedades.
- Publicada: ya se uso.
- No usar por ahora: mejor guardarla para el futuro.

## 6. Objetivo semanal

Objetivo recomendado:

**2 videos por semana**

Formula simple:

- Video 1: algo visual del juego.
- Video 2: pregunta, rareza, carta, bug gracioso o mini lore sin spoilers.

Ejemplo:

- Video visual: mostrar el cristal del Abismo.
- Video comunidad: preguntar que carta meteria la gente en el juego.

## 7. Que hacer si no hay videos nuevos

Si no hay clips nuevos, aun se puede publicar:

- una captura con texto encima;
- una pregunta a la comunidad;
- una idea de carta;
- un mini lore sin spoilers;
- una comparacion antes/despues;
- una imagen del juego con una frase;
- un bug gracioso viejo;
- un clip antiguo con texto nuevo;
- una decision de diseño explicada en corto.

No quedarse parado solo porque no haya trailer nuevo.

## 8. Como preparar un video corto

Un video corto deberia tener:

- un hook claro;
- una imagen o clip facil de entender;
- texto corto en pantalla;
- duracion aproximada de 8 a 20 segundos;
- una idea simple, no diez cosas a la vez.

Ejemplo de hook:

> Asi empieza nuestro RPG indie...

Ejemplo de texto en pantalla:

> Despiertas en el Abismo. Solo sabes que debes encontrar el portal.

## 9. Que no se debe publicar

No publicar si:

- tiene spoilers fuertes;
- promete fechas no confirmadas;
- parece anuncio oficial de algo que aun es duda;
- muestra algo roto que puede dar mala imagen sin contexto;
- no se entiende nada;
- contradice el estado real del proyecto.

Si hay duda, marcar la idea como `Necesita contexto`.

## 10. Como registrar que algo se publico

Despues de publicar, apuntar en:

`Calendario/publicaciones_realizadas.md`

Formato recomendado:

```text
Fecha:
Plataforma:
Titulo / idea:
Link:
Resultado aproximado:
Notas:
```

Ejemplo:

```text
Fecha: 2026-06-14
Plataforma: TikTok
Titulo / idea: Asi empieza nuestro RPG indie
Link: pendiente
Resultado aproximado: pendiente
Notas: usar esta idea tambien para Shorts si funciona.
```

## 11. Regla de oro

Mejor publicar algo simple, honesto y claro que esperar siempre al video perfecto.

El objetivo del banco es que el proyecto se vea vivo y que el equipo nunca se quede sin ideas.
